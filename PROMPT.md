# 手艺活记录应用代码生成提示词

## 产品思路梳理
- **定位**：苹果风格的极简移动 Web 应用，用于记录每天的手艺活次数、时间和日期。
- **主要流程**：
  1. 打开即显示当月月历，每个日期格展示日期数字与当日累计次数。
  2. 右下角显眼的圆形 "+" 按钮，一点即记，自动记录当前日期与时间。
  3. 点击某天可查看该日的时间点明细，并支持删除单条记录。
- **页面模块**：
  - 月历页：可左右切换月份，卡片式圆角格子显示日期与次数。
  - 记录按钮：苹果风格悬浮按钮，固定右下角。
  - 详情页：当日所有时间点列表，支持删除。
- **数据存储**：可选 LocalStorage、IndexedDB/SQLite 或轻量 Node/Express + SQLite。
- **UI 风格**：Apple iOS 风格，大留白、圆角卡片、柔和阴影、San Francisco 字体，9:16 竖屏适配。

## 可直接用于代码生成模型的最终 Prompt

你是一名全栈工程师，请为我生成一个可部署的 Web 应用。需求如下：

1. **产品目标**
   - 开发一个移动端优先的网页应用，用于记录用户每次手艺活的时间和日期。
   - 用户点击一个按钮即可自动生成记录。
   - 界面为 9:16 手机屏幕适配，视觉风格为苹果 iOS 极简设计。

2. **功能需求**
   1) 月历视图（主页）
      - 展示当前月份的日历（1–30/31）。
      - 每个日期格展示：日期数字、当天累计记录次数（如“3次”）。
      - 支持点击左右箭头或滑动切换月份。
      - 点击日期可进入详情页。
   2) 创建记录
      - 右下角有明显的苹果风格悬浮大圆形 “+” 按钮。
      - 点击立即创建记录，自动保存当前日期与精确时间（HH:MM）。
      - 数据写入本地 LocalStorage 或 SQLite（任选其一）。
   3) 详情页
      - 展示某一天的所有记录时间点列表。
      - 提供删除某条记录的功能。

3. **UI 风格要求**
   - 整体模仿 Apple iOS：圆角、留白、柔和阴影、SF 字体。
   - 布局适配 9:16 移动屏幕。
   - 顶部大号标题，圆角卡片式日历格子，大而明显的 “+” 按钮。

4. **技术要求**
   - 前端：HTML、CSS、JavaScript（或 React/Vue/Svelte 任意框架）。
   - 必须有移动端自适应布局。
   - 数据存储在 LocalStorage 或 SQLite（如使用 Node 后端）。
   - 给出完整的代码结构、说明、依赖、运行方式。

5. **输出格式要求**
   - 输出完整可运行的代码（前端 + 后端，如有）。
   - 样式（CSS）必须完整。
   - 说明每个文件内容。
   - 最终能在浏览器上直接访问使用。

将以上 Prompt 原样复制给代码生成模型，可直接得到完整可运行的前后端代码。
